## Day 2 Demo Commands

### Docker (show containerization)
```
docker build -t stroke-mlops:v2 .
docker run -p 8000:8000 stroke-mlops:v2
```

### Drift Detection — Clean Data (PASS)
```
python src/monitoring/drift_check.py --incoming data/processed/test.csv
```

### Drift Detection — Drifted Data (FAIL)
```
python src/monitoring/generate_drifted_data.py
python src/monitoring/drift_check.py --incoming data/incoming/drifted_patients.csv
```

### Inference Pipeline — Full Flow (drift → score → Excel)
```
python src/inference/pipeline.py --incoming data/processed/test.csv
```

### Inference Pipeline — Blocked by Drift
```
python src/inference/pipeline.py --incoming data/incoming/drifted_patients.csv
```

### CI/CD — Trigger Deploy (push code → auto-deploy to Cloud Run)
```
git add .
git commit -m "your change message"
git push origin main
```
Then watch: https://github.com/basudev2016/stroke-prediction-mlops/actions

### GCP — Manual Deployment Commands (what Terraform automates)
```
# 1. Enable APIs
gcloud services enable run.googleapis.com artifactregistry.googleapis.com cloudbuild.googleapis.com monitoring.googleapis.com logging.googleapis.com

# 2. Create Artifact Registry
gcloud artifacts repositories create stroke-mlops-repo --repository-format=docker --location=asia-south1

# 3. Configure Docker auth
gcloud auth configure-docker asia-south1-docker.pkg.dev

# 4. Tag and push image
docker tag stroke-mlops:v2 asia-south1-docker.pkg.dev/stroke-mlops/stroke-mlops-repo/stroke-api:v2
docker push asia-south1-docker.pkg.dev/stroke-mlops/stroke-mlops-repo/stroke-api:v2

# 5. Deploy to Cloud Run
gcloud run deploy stroke-api --image=asia-south1-docker.pkg.dev/stroke-mlops/stroke-mlops-repo/stroke-api:v2 --region=asia-south1 --port=8000 --allow-unauthenticated --memory=1Gi --max-instances=1

# 6. Create storage buckets
gcloud storage buckets create gs://stroke-mlops-incoming --location=asia-south1
gcloud storage buckets create gs://stroke-mlops-scored --location=asia-south1
gcloud storage buckets create gs://stroke-mlops-reports --location=asia-south1
```

### GCP — Service Account for CI/CD
```
# Create robot account
gcloud iam service-accounts create github-deploy --display-name="GitHub Deploy"

# Grant permissions
gcloud projects add-iam-policy-binding stroke-mlops --member="serviceAccount:github-deploy@stroke-mlops.iam.gserviceaccount.com" --role="roles/run.admin"
gcloud projects add-iam-policy-binding stroke-mlops --member="serviceAccount:github-deploy@stroke-mlops.iam.gserviceaccount.com" --role="roles/artifactregistry.writer"
gcloud projects add-iam-policy-binding stroke-mlops --member="serviceAccount:github-deploy@stroke-mlops.iam.gserviceaccount.com" --role="roles/iam.serviceAccountUser"

# Create key file (store as GitHub secret)
gcloud iam service-accounts keys create github-deploy-key.json --iam-account=github-deploy@stroke-mlops.iam.gserviceaccount.com
```

### Terraform (IaC — not run, shown as teaching material)
```
cd terraform
terraform init
terraform plan      # dry run — shows what would be created
terraform apply     # creates everything
terraform destroy   # removes everything
```
